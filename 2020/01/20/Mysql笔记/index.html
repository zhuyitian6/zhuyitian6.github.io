MySQL常用数据类型

数据类型是指系统中所允许的数据的类型。数据库中每个字段都应有适当的数据类型

数值类型

  • BIT[(M)]:位字段类型。M表示每个值的位数，范围1 ~ 64。如果M被省略，，默认为1

  • TINYINT[(M)]:很小的整数。带符号的范围是-128 ~ 127，无符号的范围是0 ~ 255。

  • BOOL,BLLOEAN[(M)]:是TINYINT(1)的同义词。zero值被视为假，非zero值视为真。

  • SMALLINT[(M)]:小的整数。带符号的范围是-32768 ~ 32767，无符号的范围是0~65535。

  • MEDIUMINT[(M)]:中等大小的整数。带符号的范围是-8388608 ~ 8388607，无符号的范围是0~ 16777215。

  • INT[(M)]:普通大小的整数。带符号的范围是-2147483648 ~ 2147483647，无符号的范围是0 ~ 4294967295。

  • INTEGER[(M)]:INT的同义词。

  • BIGINT[(M)]:大整数。带符号的范围是-922372036854775808 ~ 922372036854775807，无符号的范围是0 ~ 18446744073709511615。

  • DOUBLE[(M)]:普通大小(双精度)浮点数。用于存储带小数的数据

  • DECIMAL[(M,d)]:压缩的“严格”定点数。M是小数位数(精度)的总数,D是小数点(标度)后面的位数。小数点和(负号的)”-“符号不包括在M中

  • DEC:DECIMAL的同义词。

    日期和时间类型

  • DATE:日期型。支持的范围’1000-01-01’到’9999-12-31’。MySQL以’YYYY-MMMM-DD’格式显示DATE值，但允许使用字符串或数字为DATE列分配值。

  • DATETIME:日期与时间的结合。支持的范围’1000-01-01 00:00:00’到’9999-12-31 23:59:59’。MySQL以’YYYY-MMMM-DD HH:MM:SS’格式显示DATE值，但允许使用字符串或数字为DATETIME列分配值。

  • TIMESTAMP[(M)]:时间戳。范围是’1970-01-01 00:00:00’到2037年。TIMESTAMP列用于INERT或UPDATE操作时记录日期和时间，如果不分配一个值，表中的第一个timestamp列自动设置为最近操作的日期和时间。

  • TIME:时间型。范围是’-835:59:59’到’838:59:59’。MySQL以’HH:MM:SS’格式显示显示TIME值，但允许使用字符串或数字为DATETIME列分配值。

  • YEAR[(2|4)]:两位或四位格式的年。默认是四位格式。在四位格式中，允许的致使1901

    2155和0000。在两位格中，允许的值70—

    69，表示从1970~2069年

    字符串类型

  • CHAR[(M)]或CHARACTER[(M)]:固定长度的字符数据类型。用于保存以为本格式存储的信息。M表示长度，其范围是0~255个字符。

  • VARCHAR[(M)]:可变长度的字符数据类型。用于保存以为本格式存储的信息。M表示长度，其范围是0~65535个字符。

  • TINYTEXT:最大长度位255(2^(8-1))字符的TEXT列。

  • TEXT[(M)]:最大长度位65535(2^(16-1))字符的TEXT列。

数据完整新约束

主键

表示某一列或某些列构成的一个组合,多个列组合而成的主键称为复合主键,主键遵循的规则如下：
每一个表只能定义一个主键

主键的值( 键值 )
能够唯一标识表中的每一行记录,且不能为NULL,即表中两条不同的记录在主键不能具有相同的值,这是唯一性规则
复合主键不能包含不必要的多余列
即当从一个复合主键中删除一列后,如剩下的列依然复合唯一性原则,那么这个复合主键是不正确的
一个列名在复合主键的列表中只能出现一次
定义主键约束后
数据库会自动为主键创建一个唯一索引,用于查询中使用主键对数据进行快速检索,该检索名的默认俄日primary,也可以重命名

创建主键约束

create table 语句中使用关键字primary key来实现
列级完整性约束方式定义主键
即在某个字段后加关键字primary key
表级完整性约束方式定义主键并指定主键约束名称
适合定义多列所构成的主键,语法
constraint 约束名 primary key(列名1,……列名n);

**添加主键约束
Alter table语句中使用关键字Primary key来实现

定义完数据表后添加主键约束。

语法：

删除主键约束时,表中只能定义一个主键,无论逐渐约束有没有名称,都使用以下用法：

在MySQL中,可通过关键字auto_increment为列设置自增属性,只有个整型列才能设置此属性,每个表只能定义一个auto_increment列,并且必须在该列上定义主键约束 (Primary key)或候选键(Unique)

默认值约束：

  • 创建表时,给字段增加默认值约束,方法如下：

    CREATE TABLE Persons
(
Id_P int ,
LastName varchar(10) ,
FirstName varchar(10),
Address varchar(100),

    以上创建的表person中,对城市‘City’给了默认值,在插入数据的时候如果不输入其他值,给一个default,mysql会自动给默认值’Shanghai’

  • 删除表中某个字段的默认值,方法如下：

表单查询

选择字段

从mysql数据库表中,选择指定的字段进行查询,语法格式下
select 目标列表达式1,目标列表达式2,...目标列表达式n
from  表名;
查询所有字段
有两种方法

  • 方法一

    在select关键字后面列出所有字段,字段顺序不要求,语法格式如下
    select 目标列表达式1,目标列表达式2,...目标列表达式n
from  表名;

  • 方法二

    在select 关键字后面直接使用星号(*)通配符,不必列出表中所有的列名,此时查询的结果集中各列的次序与表定义中的列的次序一致,语法格式如下
    select * from  表名;
    查询时去掉重复记录
在字段前加distinct或distinctrow,具体用法如下

  • 用法一

    查询表中所有记录,但不展示重复记录,语法格式如下
    select  distinct  * from table;
select  distinctrow * from table;

  • 用法二

    查询表中指定的列,但不展示重复记录,语法格式如下
    select  distinct目标列表达式1,目标列表达式2,...目标列表达式n
from  表名;
或:
select  distincrow目标列表达式1,目标列表达式2,...目标列表达式n
from  表名;
    该语句会查询出classNo,department两个字段的不重复的记录,distinct会对出现在它后面的所有目标列都产生作用,而不是第一个目标了

查询经过计算的值

在目标列表达式中,不仅可以是表中的字段,还可以对目标列进行四则运算(加+,减-,乘*,除/,求余),方法如下
select  目标列,format(目标列计算,[小数位数])from  表名;
例子：有如下表单:tb_fruit,分别有name,price,num表示水果名,单价价格,数量,如果要从tb_fruit中选择水果,单价,数量,以及每种水果的总价格,我们可以这么样查询
select name,price,num,format(price*num,2) from tb_fruit;

查询字段同时定义字段名称

在系统中输出查询结果集中某些列或所有列的名称时,若希望这些列的名称显示为自定义的列名,则可以在select自居的目标列式之后添加AS子句,格式如下

排序查询

  • 升序排序

    默认可以不加asc,例子
    select studentName,native,nation from tb_student order by studentName;

  • 降序排序

    需在字段后加desc,例子
    select * from tb_score where score>85 order by score desc;

限制查询结果的数量

只查看结果集中想要的行的记录,语法格式如下：

  • 方法一 Limit[位置偏移量,] 行数其中,行数是查询结果中返回的记录数,位置偏移量是一个可选参数,是指mysql从哪一行开始显示。第一条记录也的位置偏移量是0,第二条是1,以此类推,如果不指定位置偏移量,系统会自动从表中第一条记录开始显示,例

    select studentNo,courseNo,score from tb_score 
order by score desc
Limit 2,3```

该sql语句是查询成绩排名第3至5名的学生学号、课程编号和成绩。

  • 方法二：limit 行数 offset 位置偏移量,例

    select studentNo,courseNo,score
from tb_score 
order by score desc
Limit 3 offset 2;

注意：如果select语句中既有order by语句,又有limit子句,则limit子句必须在order by子句的后面,否则将报错

连接查询和联合查询

交叉查询

交叉查询又称为笛卡尔积,即把一张表的每一行与另一张表的每一行连接起来,返回两张表的每一行连接的所有可能的搭配结果,器连接的结果是两张表的记录数行数的乘积,应结合过滤条件where使用,较少返回的结果集,该查询使用较少,语法结果如下
select * from 表1 cross join 表2
例子：select * from tb_class cross join tb_student;
或者
select *  from 表1,表2;
例子：select * from tb_class,tb_student;

内连接查询

内连接是通过设置连接条件移除交叉查询结果集中某些数据行。也就是说,内连接就是使用比较运算符进行表间某些自段子的比较操作,并将与条件相匹配的数据行组成新的记录,其目的就是为了消除交叉连接中某些没有意义的数据行也就是说,只有满足条件的记录才能出现在结果集中`
`语法格式1
select  目标列表达式1,目标列表达式2,...,目标列表达式n 
from 表1  [inner] join表2
on 连接条件
[where 过滤条件];
或者`
`语法格式2
select  目标列表达式1,目标列表达式2,...,目标列表达式n 
from 表1 ,表2
Where 连接条件 [and 过滤条件];
例子：查询学生的学号,姓名以及班级名称
select studentNo,studentName,className 
from tb_student inner join tb_class 
on tb_student.classNo=tb_class.classNo
Where nation=’汉’;
或者
select studentNo,studentName,className 
from tb_student,tb_class 
Where tb_student.classNo=tb_class.classNo
and nation=’汉’;
等值与非等值连接
连接查询中用来连接两个表的条件为连接条件,其一般格式是：[<表名1.>]<字段名1> <比较运算符> [<表名2.>]<字段名2>`
`其中比较运算符有：=、>、<、>=、<=、！=(<>).当比较运算符为‘=’时,表时等值连接,其他运算符为非等值连接
给表取别名
当表的名称很长或需要多次使用相同的表时,可以为表指定别名,用别名来代表原来的表名,如果在from子句中制定了表别名,那么它所在的select语句的其他子句都必须ongoing表命名来代替原来的别名
语法格式:表名 [as] 表别名,其中,as为可选项,使用时可以隐藏
自连接
若某个表与自身进行连接,称为自表连接或自身连接,简称自连接。使用自连接时,需要为表指定多个不同的别名,且对所有查询字段的引用均必须使用表别名限定,否则select操作会失败
自然连接
自然连接(natural join)只有当连接字段在两张表中的字段名都相同时才可以使用,否则返回的是笛卡尔积的结果集,其关键字为natrual join`
`语法格式
select  目标列表达式1,目标列表达式2,...,目标列表达式n 
from 表1 natrual join表2;

外连接

左外连接
左外连接,也称为左连接(left outer join或left join),用于返回该关键字左边表(基表)的所有记录,并用这些记录与该关键字右边表(参考表)中的记录进行匹配,如果左边的某些记录在右边表中没有匹配的记录,就和右表中的万能行连接,即右边表对应的字段值均被设置为空值null`
`语法格式
select  目标列表达式1,目标列表达式2,...,目标列表达式n 
from 表1 left outer join表2
on 连接条件
[where 过滤条件];
右外连接
右外连接,也称右连接(right outer join或right join),以右表为基表,其连接方法与左外连接完全一样,即返回右表的所有记录,并用这些记录与左边表(参考表)中的记录进行匹配,如果右表的某些记录在左表中没有匹配的记录,左表对应的字段值均被设置为空值null`
`语法格式
select  目标列表达式1,目标列表达式2,...,目标列表达式n 
from 表1 right outer join表2
on 连接条件
[where 过滤条件];

联合查询

使用union关键字可以把来自多个select语句中的结果组合到一个结果集中。这种查询方式称为并(union)运算或联合查询。合并时,多个select子句中对应的字段数和数据类型必须相同。语法格式如下
Select 目标表达式 from  表1,表2
Where 连接条件[and 过滤条件]
Union[all]
Select 目标表达式 from 表1,表2 
Where 连接条件[and 过滤条件];

子查询

查询条件比较相等 (=)

例：从tb_fruit表中,查看名字为苹果的记录：

查询条件判断不相等(<>或者！=)

例：从tb_fruit表中,查看名字不是苹果的记录
SELECT * FROM tb_fruit where name！='苹果';

SELECT * FROM tb_fruit where name<>'苹果';

例：从以上的tb_fruit表中,查看id>2的记录

判断完全相等(<=>)

与 = 的区别在于当两个操作码均为 NULL 时,其所得值为 1 而不为NULL,而当一个操作码为 NULL 时,其所得值为 0而不为 NULL`
`例子,从tb_fruit表中,查看名字是苹果的记录

查询条件判断大于等于>=

例：从以上的tb_fruit表中,查看id>=2的记录

查询条件判断小于<

例：从以上的tb_fruit表中,查看id<2的记录
SELECT * FROMtb_fruit where id<2;

查询条件判断小于等于<=

例：从以上的tb_fruit表中,查看id<=2的记录
SELECT * FROM tb_fruit where id<=2;

Between…. and关键字的查询

Not Between…. and关键字的查询

带like关键字的字符串匹配查询,like关键字在查询中相当于模糊查询,它语法如下

使用like语句时,通常会结合”%”或者”_”这两种通配符进行使用。%代表任意长度的字符,“_”代表任意单个字符,能进行运算匹配的字段可以是char,varchar,text,datetaime等等数据类型。

视图

视图的特点

  • 视图是由基本表(实表)产生的虚表
  • 视图的创建和删除不影响基本表
  • 视图的列可以来自于不同表,是标的抽象和逻辑意义上的关系
  • 视图的内容的添加、删除和修改会影响基本表(来自同一张基本表)
  • 视图来自多个基本表,不允许添加和删除数据,可以进行修改
  • 视图的查询语句不能包括from字句中的子查询
  • 与视图相关的表或视图被删除,试图将不能再使用

创建视图

基本语法
create view 视图名 as 查询语句;

例：数据库JXGL,包括表students,score,course,departments

创建视图VIEW_CJ,包括学号、课程名和成绩字段

Create view view_cj As
select s_no 学号,c_name 课程名,report 成绩 from 
course natural join score;

创建视图VIEW_CJ_TJ,统计包括某位学生所有课程的总成绩、平均成绩。

Create view view_cj_tj As
select 学号, sum(成绩) 总成绩,avg(成绩) 平均成绩 from view_cj
group by 学号;

创建视图VIEW_AVG,统计各门课程平均成绩,并按课程名称降序排列。

Create view VIEW_AVG as
select 课程名,avg(成绩) from view_cj
group by 课程名 order by 课程名 desc;

修改视图VIEW_STU,用AS定义列别名,并增加“人数”字段,统计各系男、女生人数

Create or replace view  VIEW_STU as
select  D_No ,sex ,count(*)  人数 from students 
group by D_No,sex;

修改视图

Alter view 视图名 As 查询语句;
或:
Create or replace view 视图名 As 查询语句;(既可以用来创建也可以修改)

触发器：数据库的对象之一,通过触发时间来及或执行事件的运行

触发器的创建

Delimiter //    --定义结束符,';'起到逗号作用

INSERT触发器

例：在数据库db_school的表tb_student中重新创建触发器in_trigger,用于每次向表tb_student中插入一行数据时将学生的变量str的值设置为新插入的学生的学号
触发时刻：在表tb_student中插入一行数据之后,触发事件之后After
Create trigger in_trigger 
After 
Insert on tb_student for each row
set @str=new.studentNo;

Update触发器

例：在数据库db_school的表tb_student中创建触发器up_trigger,用于每次更新表tb_student时,将该表中nation列的值设置为native列的值
触发时刻: 获取更新之前的数据(old.native),触发时刻就是before

对于每个列来说：
old.字段 更新tb_student之前的数据
new.字段 更新tb_student之后的数据

Create trigger up_trigger
before
update on tb_student for each row
Set new.nation=old.native;

Delete触发器

例：在数据库db_school的表tb_student中创建触发器tri_d,用于每次删除表tb_student时记录将学生变量str设置为删除学生的学号

Create trigger tri_d
before

set @str=old.studentNo;

4.对同一张表可以创建最多六种不同时刻的触发器

事件

Mysql服务器存在功能模块--事件调度器
  • 在指定的时刻才被执行的某些特定任务就是事件,这些特定任务通常是一些确定的SQL语句
  • 事件调度器可以在指定的时刻执行某些特定的任务,并可以取代原先只能由操作系统的计划任务来执行的工作
  • Mysql的事件调度器可以精确到每秒钟执行一个任务,对于一些对数据实时性要求较高的应用非常适合,例如股票、赔率和比分等

触发器和事件的区别：

事件是基于特定时间周期触发来执行某些任务
触发器是基于某个表所产生的事件触发的

创建事件

Delimiter $$

Create event if not exists 事件名称
On Schedule 时间调度
Do
Begin
事件语句1;
事件语句2;
end
$$

时间调度(Schedule)：用于指定事件何时发生或者每搁多久发生一次

时间调度的语法
AT timestamp [+INTERVAL interval]...

|EVERY interval

               [STARTS timestamp [+INTERVAL interval]...]

               [ENDS timestamp [+INTERVAL interval]...]

interval的语法：
               quantity|YEAR|MONTH|WEEK|DAY|HOUR|MINUTE|SECOND|
                        YEAR_MONTH|DAY_HOUR|DAY_MINUTE|
                        DAY_SECOND|HOUR_MINUTE|HOUR_SECOND|MINUTE_SECOND|
timestamp--表示具体的时间点,后面可以加上一个时间间隔,表示在这个时间间隔后事件发生
interval--表示这个时间间隔,由数值和单位构成,例如：1 MONTH`
`quantity--间隔时间的数值

操作开启事件

查看事件调度器是否开启

SHOW VARIABLES Like 'EVENT_SCHEDULER';

查看系统变量：Select @@EVENT_SCHEDULER;

开启事件调度器
Set Global EVENT_SCHEDULER=1;
或者
Set Global EVENT_SCHEDULER=True;
关闭事件调度器
Set Global EVENT_SCHEDULER=0;

例:在数据库db_event中创建一个表tb_test实现每秒插入一个条记录

$$

存储过程

概念：由SQL语句集合组成,SQL语句集合包含声明式语句、过程式语句,创建的存储过程在数据库不用再次编译,直接调用

存储过程

是一组为了完成某特定功能的SQL语句集,其实质上就是一段存放在数据库中的代码,它可以由声明式语句(如CREATE、UPDATE和SELECT等语句)和过程式语句(如IF-THEN-ELSE控制结构语句)组成
这组语句经过编译后会存储在数据库中,用户只需通过指定存储过程的名字并给定参数(如果该存储过程带有参数),即可随时调用处理它,而不必重新编译,提高数据库操作语句的执行效率
优点：
  • 增强SQL语言的功能与灵活性。
  • 良好的封装性。
  • 高性能。
  • 可减少网络流量。

创建存储过程

Create procedure 存储过程名(参数)
存储过程体;
执行存储过程
call 存储过程名(@参数)

参数：IN|OUT|INOUT 参数名称 数据类型

  • 输入参数：IN 参数名称 数据类型

    delimiter $
Create procedure pro_in(IN p_in int)
begin
select p_in;  
set p_in=2;
select p_in;  
end
$

输入参数：只受外部输入值的影响,不会受存储过程中程序影响

  • 输出参数：OUT 参数名称 数据类型

    delimiter $
Create procedure pro_out(OUT p_out int)
begin
select p_out;  
set p_out=2;
select p_out;   
end
$

输出参数：不受外部输入值的影响,只会受存储过程中程序影响,返回存储过程中的值给输出参数

输入输出参数：INOUT 参数名称 数据类型

delimiter $
Create procedure pro_io(INOUT p_io int)
begin
select p_io;  
set p_io=2;
select p_io;  
end
$

**输入输出参数：受外部输入值的影响,又会受存储过程中程序影响,返回存储过程中的值给输入输出参数;

例题1：在数据库db_school中创建一个存储过程,用于实现给定表tb_student中一个学生的学号即可修改表tb_student中该学生的性别为一个指定的性别(使用该存储过程修改学生表中的学号、性别)

输入参数
给定学号 IN sno int 指定性别 IN ss char(1)

Create procedure p_st( IN sno int,IN ss char(1))
update tb_student  set sex=ss  where studentNo=sno;

call p_st(18030330,'男');

局部变量

**如何定义局部变量：

声明局部变量,并同时赋默认值
Declare  变量名称 数据类型 default 默认值;

注意

  • 局部变量只能在存储过程体的BEGIN…END语句块中声明。
  • 局部变量必须在存储过程体的开头处声明。
  • 局部变量的作用范围仅限于声明它的BEGIN…END语句块,其它语句块中的语句不可以使用它。
  • 局部变量不同于用户变量,两者的区别是：局部变量声明时,在其前面没有使用“@”符号,并且它只能声明它的BEGIN…END语句块中的语句所使用;用户变量在声明时,会在前面使用“@”符号,同时声明的用户变量存在于整个会话之中。
代码示例`
`全局变量x3,x4,局部变量x1,x2,设置x3的值等于x1,x4等于x2,定义x1为字符类型设置值为1,定义x2字符类型0
DELIMITER $$
CREATE PROCEDURE proc4(out x3 int,out x4 int)  
begin   
 declare x1 int default 1;  
  begin    
  declare x2 int default 0;  
  select x2;   
  set x4=x2;  
  set x2=x1;  
  select x4;   
  select x2;    
  end; 
select x1;    
set x3=x1;  
select x3;     
set x1=x2;  
select x1;    
end;  
$$  

DELIMITER ;

游标(CURSOR)

游标(cursor)就是游动的标识,通俗的这么说,一条sql取出对应n条结果资源的接口/句柄,就是游标,沿着游标可以一次取出一行
  • 游标的使用场景
    • 只能用于存储过程或存储函数中,不能单独在查询操作中使用;
    • 在存储过程或存储函数中可以定义多个游标,但是一个BEGIN…END语句块中每一个游标的名字必须是唯一的;
    • 游标不是一条SELECT语句,是被SELECT语句检索出来的结果集;
游标的使用步骤

口令：申开取关

申明游标

Declare 游标名称 CURSOR for Select 语句;

开游标

OPEN 游标名称;

游标取值

FETCH 游标名 INTO 变量名;

关闭游标

CLOSE 游标名称;

句柄

定义句柄

Declare continue handler for NOT FOUND
set 变量名=0|false;
例1：使用游标逐行取category表中三行数据
delimiter $
create procedure p1()
begin
declare c_id int default 0;
declare p_id int default 0;
declare c_name varchar(20);
Declare c_ga CURSOR for Select  cat_id, cat_name,parent_id
from category;
OPEN c_ga;

FETCH c_ga INTO c_id,c_name,p_id;
select c_id,c_name,p_id;

FETCH c_ga INTO c_id,c_name,p_id;
select c_id,c_name,p_id;

FETCH c_ga INTO c_id,c_name,p_id;
select c_id,c_name,p_id;
CLOSE c_ga;
end
$
例2：使用循环和游标逐行取category表中事十五行数据
delimiter $
create procedure p4()
begin
declare i int default 0;
declare c_id int default 0;
declare p_id int default 0;
declare c_name varchar(20);
declare cn int;
Declare c_ga CURSOR for Select  cat_id, cat_name,parent_id
from category;
Select count(*) into cn from category;
OPEN c_ga;
while i<cn  Do
FETCH c_ga INTO c_id,c_name,p_id;
select c_id,c_name,p_id;
FETCH c_ga INTO c_id,c_name,p_id;
set i=i+1;
end while;
CLOSE c_ga;
end
$
例3：使用句柄和游标逐行取category表中十五行数据
delimiter $
create procedure p9()
begin
declare i int default 0;
declare c_id int default 0;
declare p_id int default 0;
declare c_name varchar(20);
declare search int default 1;
Declare c_ga CURSOR for Select  cat_id, cat_name,parent_id
from category;
Declare continue handler for NOT FOUND
set search=0;
OPEN c_ga;
FETCH c_ga INTO c_id,c_name,p_id;
while search  Do
select c_id,c_name,p_id;
FETCH c_ga INTO c_id,c_name,p_id;
end while;
CLOSE c_ga;
end
$

用户的创建和赋权

创建用户账号

crate user 用户名@localhost|IP地址;

删除用户

drop user 用户名@'IP地址'

修改用户账号

可以使用rename user语句修改一个或多个已经存在的MySQL用户账号，其语法格式如下
rename user old_user to new_user [,old_user to new_user];

修改用户口令

set password=password("新密码");
或
use mysql;

select user,host,password from user;
update user set password=PASSWORD('123')  where user='root';

flush privileges;
查看连接用户名
select user,host from mysql.user;

刷新权限

flush privileges;
重新登陆
mysql -u 新连接名 -h 新IP -p:
password：

给新用户授权

grant 权限1,权限2... on 数据库名.* to '用户名'@'IP地址'；

给所有权限给新建的用户

grant all privileges on *.* to '用户名'@'IP';

授权同时创建新用户

grant select,delete,update,insert on 数据库名.* to '新用户名'@'新IP' identified by '密码';

查看用户权限

show grants for '用户名'@'IP地址' \G

权限的撤销

revoke 权限1,权限2...on *[数据库名].*[表名] from '新连接名'@'IP';

或

revoke all privileges,grant option
from user@IP[localhost]
